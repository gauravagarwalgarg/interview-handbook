**Focus: CMakeLists.txt structure, cross-compilation toolchains, library management, build targets, integration with Yocto SDK**

## ⚡ Quick-Fire (Build System Gauge)

| Question | Expected Answer |
|----------|----------------|
| What is CMake vs Make? | CMake: meta-build system (generates Makefiles/Ninja files). Make: executes build rules. |
| What is Ninja vs Make? | Ninja: faster parallel builds, simpler design, no user-facing features. Generated by CMake. |
| What is a CMake target? | Named entity (executable, library) with properties (sources, includes, link deps). |
| What does `target_link_libraries(PRIVATE)` mean? | Dependency is used for building this target but NOT propagated to consumers. |
| PUBLIC vs PRIVATE vs INTERFACE? | PUBLIC: used by target + consumers. PRIVATE: only this target. INTERFACE: only consumers. |
| What is a toolchain file? | CMake file defining compiler, sysroot, and system info for cross-compilation. |
| How does Yocto SDK integrate with CMake? | SDK sets `CMAKE_TOOLCHAIN_FILE` via environment or `OEToolchainConfig.cmake`. |
| What is `find_package` vs `FetchContent`? | `find_package`: finds pre-installed libs. `FetchContent`: downloads and builds at configure time. |
| What is CTest? | CMake's test runner  discovers and runs tests, integrates with CDash. |
| What is `install(TARGETS ...)` for? | Defines where built artifacts go during `cmake --install` (or Yocto `do_install`). |

## 🔬 Q1: Production CMakeLists.txt for Embedded Application

**Question:** Write a CMakeLists.txt for a sensor daemon that:
- Has a library (`libsensor`) and an executable (`sensor-daemon`)
- Links against protobuf, libgpiod, and a custom FPGA access library
- Supports cross-compilation via toolchain file
- Has unit tests with Google Test
- Installs binary + systemd service + config file
- Exports version from Git tag

**Expected answer:**

```cmake
cmake_minimum_required(VERSION 3.20)

# Version from git
execute_process(
    COMMAND git describe --tags --always
    WORKING_DIRECTORY ${CMAKE_SOURCE_DIR}
    OUTPUT_VARIABLE GIT_VERSION
    OUTPUT_STRIP_TRAILING_WHITESPACE
)
project(sensor-daemon VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)  # For clang-tidy

# Options
option(BUILD_TESTS "Build unit tests" ON)
option(ENABLE_SANITIZERS "Enable ASan/UBSan" OFF)

# Find dependencies
find_package(Protobuf REQUIRED)
find_package(PkgConfig REQUIRED)
pkg_check_module(GPIOD REQUIRED libgpiod)

# Sanitizers (development builds)
if(ENABLE_SANITIZERS)
    add_compile_options(-fsanitize=address,undefined -fno-omit-frame-pointer)
    add_link_options(-fsanitize=address,undefined)
endif()

# --- Library: libsensor ---
add_library(sensor STATIC
    src/sensor/imu_driver.cpp
    src/sensor/temp_driver.cpp
    src/sensor/fusion_engine.cpp
)
target_include_directories(sensor
    PUBLIC  $<BUILD_INTERFACE:${CMAKE_SOURCE_DIR}/include>
            $<INSTALL_INTERFACE:include>
    PRIVATE ${CMAKE_SOURCE_DIR}/src
)
target_link_libraries(sensor
    PUBLIC  protobuf::libprotobuf
    PRIVATE ${GPIOD_LIBRARIES}
)
target_compile_definitions(sensor
    PRIVATE APP_VERSION="${GIT_VERSION}"
)

# --- Executable ---
add_executable(sensor-daemon src/main.cpp)
target_link_libraries(sensor-daemon PRIVATE sensor)

# --- Install ---
include(GNUInstallDirs)
install(TARGETS sensor-daemon RUNTIME DESTINATION ${CMAKE_INSTALL_BINDIR})
install(FILES deploy/sensor-daemon.service
    DESTINATION ${CMAKE_INSTALL_PREFIX}/lib/systemd/system)
install(FILES deploy/sensor.conf
    DESTINATION ${CMAKE_INSTALL_SYSCONFDIR}/sensor-daemon)

# --- Tests ---
if(BUILD_TESTS)
    enable_testing()
    find_package(GTest REQUIRED)
    
    add_executable(test_fusion tests/test_fusion.cpp)
    target_link_libraries(test_fusion PRIVATE sensor GTest::gtest_main)
    
    add_test(NAME FusionTests COMMAND test_fusion)
    
    # Coverage target
    if(CMAKE_BUILD_TYPE STREQUAL "Debug")
        target_compile_options(test_fusion PRIVATE --coverage)
        target_link_options(test_fusion PRIVATE --coverage)
    endif()
endif()
```

**Cross-compilation toolchain file:**

```cmake
# toolchain-aarch64-linux.cmake
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR aarch64)

set(CMAKE_SYSROOT $ENV{SDKTARGETSYSROOT})
set(CMAKE_C_COMPILER $ENV{CC})
set(CMAKE_CXX_COMPILER $ENV{CXX})

set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_PACKAGE ONLY)
```

**Follow-up probes:**
- What is `$<BUILD_INTERFACE:...>` vs `$<INSTALL_INTERFACE:...>`? (Generator expressions  different include paths for building vs installed SDK)
- How do you prevent host contamination in cross-builds? (`CMAKE_FIND_ROOT_PATH_MODE_*` = ONLY)
- How does this integrate with Yocto? (`inherit cmake` in recipe, `EXTRA_OECMAKE` for options)

## 🔬 Q2: Modern CMake  Target-Based Design

**Question:** Your project has this structure. Design the CMake targets with proper dependency propagation:

```
platform/
├── libs/
│   ├── hal/          (hardware abstraction  depends on nothing)
│   ├── protocol/     (CAN/UART protocol  depends on hal)
│   └── middleware/   (REST/MQTT client  depends on protocol + external libs)
├── apps/
│   ├── daemon/       (main app  depends on middleware)
│   └── cli-tool/     (debug tool  depends on hal only)
└── tests/
    └── ...
```

**Expected answer:**

```cmake
# libs/hal/CMakeLists.txt
add_library(hal STATIC src/gpio.cpp src/spi.cpp src/uart_hw.cpp)
target_include_directories(hal PUBLIC include/)

# libs/protocol/CMakeLists.txt
add_library(protocol STATIC src/can_parser.cpp src/uart_framing.cpp)
target_include_directories(protocol PUBLIC include/)
target_link_libraries(protocol PUBLIC hal)  # Consumers also need hal headers

# libs/middleware/CMakeLists.txt
add_library(middleware STATIC src/mqtt_client.cpp src/rest_client.cpp)
target_include_directories(middleware PUBLIC include/)
target_link_libraries(middleware
    PUBLIC protocol               # Consumers need protocol headers
    PRIVATE OpenSSL::SSL curl     # Implementation detail  not propagated
)

# apps/daemon/CMakeLists.txt
add_executable(sensor-daemon main.cpp app.cpp)
target_link_libraries(sensor-daemon PRIVATE middleware)
# Transitively gets: middleware → protocol → hal (all PUBLIC links propagate)

# apps/cli-tool/CMakeLists.txt
add_executable(cli-tool main.cpp)
target_link_libraries(cli-tool PRIVATE hal)  # Only needs hal, not middleware
```

**Key insight:** PUBLIC/PRIVATE on `target_link_libraries` controls transitive dependency propagation. This is the foundation of modern CMake  treat targets as the unit of abstraction, not variables.

## ⚡ CMake Anti-Patterns to Probe

| Anti-Pattern | Why It's Bad | Correct Approach |
|-------------|-------------|-----------------|
| `include_directories(...)` (global) | Pollutes all targets | `target_include_directories(... PRIVATE/PUBLIC)` |
| `link_libraries(...)` (global) | All targets link everything | `target_link_libraries(... PRIVATE)` |
| `set(CMAKE_CXX_FLAGS ...)` globally | Affects all targets | `target_compile_options(... PRIVATE)` |
| `file(GLOB ...)` for sources | Doesn't detect new files without re-cmake | Explicit source lists |
| `add_definitions(-DFOO)` | Global | `target_compile_definitions(... PRIVATE FOO)` |
